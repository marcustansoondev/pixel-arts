# Pixel Museum Setup Instructions

This document explains how to reconstruct this interactive Pixel Museum web application gallery from scratch, generate the 135 procedural animal sprites, 127 fruit sprites, 200 vegetable sprites, 270 kitchen sprites, 350 vehicle sprites, 300 electronics sprites, 300 clothing sprites, 202 weather sprites, and 200 building sprites (totaling 2084 items), and host it locally in a new environment or chat session.

---

## Quick Start: Restoring in a New Chat Session

If you are importing this project into a new chat session, simply upload `pixel_zoo.zip` and run the following command to restore all files (HTML/CSS/JS code, directory structures, PNG images, and Python generator scripts) instantly:

```bash
unzip -o pixel_zoo.zip
```

---

## Step 1: Create the Web App Frontend

If you are rebuilding the frontend code manually:

1. **`index.html`**: Structures the dashboard page layout with a glassmorphic sidebar panel, search/filter bars, a responsive item grid, and a pixel-art details modal containing magnification controls and palette emulation shaders.
2. **`styles.css`**: Embeds typography, defines dynamic background blur glow orbs, configures custom themes (Midnight Purple, Cyberpunk Neon, Matrix Green, Classic Dark), and implements strict `image-rendering: pixelated;` rules to render retro sprites with crisp block edges.
3. **`app.js`**: Controls dataset configurations for the 1382 items, responsive keyword search, category filters (Animals, Fruits, Vegetables, Kitchen, Vehicles, Electronics, Clothing), global slider scaling, modal trigger loops, canvas filter application, and accessibility focus trapping.

---

## Step 2: Generate all 932 Sprites

To programmatically generate all 932 retro 50x50 PNG sprites and automatically build the database:

*(Note: Do not delete the `images/` directory before generation, as it contains essential base sprites that are not procedurally generated!)*

0. **Install Python Image Library (Pillow):**
```bash
pip install Pillow
```

1. Run `generate_all.py` in the workspace to execute the base animal pipeline (generating the first 85 animal sprites and the first 52 fruit sprites, resolving derived layers, and updating the database):
```bash

```

```bash

python3 generate_twemoji_animals.py
```

## Step 3: Run the Web Server

Start a background Python HTTP server on port 8080 to serve the directory:
```bash
python3 -m http.server 8080
```
Open your browser or Web Preview on port 8080 to view, search, scale, and filter all 1784 items.

---

## Step 4: Package the Files into a ZIP Archive

Run the following command to compress the HTML page, CSS styles, JavaScript logic, generator scripts, instructions text, and all sprites together:
```bash
python3 -c "import zipfile, glob; zipf = zipfile.ZipFile('pixel_zoo.zip', 'w'); [zipf.write(f) for f in ['index.html', 'styles.css', 'app.js', 'instructions.txt'] + glob.glob('*.py') + glob.glob('images/animals/*.png') + glob.glob('images/fruits/*.png') + glob.glob('images/vegetables/*.png') + glob.glob('images/kitchen/*.png') + glob.glob('images/vehicles/*.png') + glob.glob('images/electronics/*.png') + glob.glob('images/clothing/*.png') + glob.glob('images/weather/*.png') + glob.glob('images/buildings/*.png')]; zipf.close()"
```
This produces `pixel_zoo.zip` which contains the complete museum.
